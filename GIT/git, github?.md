# Git, GitHub이란?
- 내가 원하는 시점마다 깃발을 꽂고, 깃발이 꽂힌 시점으로 자유롭게 이동할 수 있도록 해 주는 소스 코드 버전관리 시스템이 **Git**이다.
- Git을 관리하는 프로젝트를 올려둘 수 있는 대표적인 Git 호스팅 사이트 중 하나가 바로 **GitHub**이다.
- 블로그 -> 네이버, 티스토리, 벨로그,... => Git으로 관리하는 프로젝트를 올릴 수 있는 사이트도 GitHub뿐 아니라 GitLab, BitBucket 등 다양하다.

# 과제1 - commit과 push란?
- `git add`를 실행하면 내가 추가한 파일들이 스테이징 상태가 된다.
- `git commit`은 스테이징 상태가 된 파일들을 스냅샷 형태로 남긴다고 보면 된다.
  - 보통 어떤 변경 사항이 있는지 메시지와 함께 기록하는 것이 일반적이며 이러한 특성상 커밋을 변경 사항에 나눠 하는 것이 이상적이다.
- `git push`는 로컬 저장소(.git)에 저장된 변경된 파일들을 github과 같은 호스팅 사이트에 업로드하는 것이다.
  - 한번 푸시하게 되면 어디서든 `git clone`을 통해 레포지토리를 불러올 수 있다.
